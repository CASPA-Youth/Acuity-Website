Two Competition Categories
* Mental Health (up to them, more freedom of design)
   * Create a consumer-based application to promote mental wellness, foster good habits, or to help people mentally in their daily lives
   * Teams should use various technical measures to ensure this (not just simply tracking people’s journals or something). Complexity points is awarded to creative application of psychology and whatnot
* Machine Learning & Agents
   * Engineer an ai tool or use agents to automate one aspect of a usually difficult workflow. 
   * Teams can create things like a custom IDE environment with their own version of a code auto-filler, a script that reads emails and turns them into interactive checklists. 
   * This can be a lot more saturated and uncreative in ideas, but it’s all about execution and which teams are willing to tackle harder ideas.
* Randomized Simulation (more tightly controlled, appeals to those who want raw data sci)
   * Create a real-time randomized simulation for real-life data scenarios, like supply chains and trucking. All scenarios must contain a source, transit, and ending.
   * Simulate realistic data streams via various randomization methods to simulate real-life complexities and losses. Create a cool UI to represent this data simulation, such as adding 3D visualizations, creative data visualizations, toggles to change randomness, etc. 
   * The design is where randomness is
